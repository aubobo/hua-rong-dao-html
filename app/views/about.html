About the game

华容道 is one of my favorite board games when I was a kid, I used to spend the whole day trying to solve just one level. Now I am an UI engineer, so why not ask the computer to help me out? Also, in each step where I get stuck, wouldn't it be great if the computer can take it from there and help me solve the puzzle? And that's why I made this application.

Frameworks

Good frameworks can help scaffold and build HTML5 project easily, I choose Yeoman because it integrates tools like Grunt, Bower etc and it also works smoothly with AngularJS.

The above frameworks are super awesome for managing front-end packages and libraries, they are like the assembly line which can efficiently improve the production. But where to find the workers? There's no doubt that AngularJS and Bootstrap are the most popular front-end libraries these days. AngularJS has so many powerful features that I didn't expect to see in JavaScript world, for example, data binding, services, factories and custom directives, filters etc. Bootstrap, in the other hand, provides easy grid layouts, a variety of componenets that normally will take us amount of time to implement by ourselves.

Algorithm

Normally in order to solve this puzzle game, we have to use BFS(Breadth-First Search) and DFS(Depth-First Search). Let's talk about BFS first, from Wikipedia, "BFS is an algorithm for traversing or searching tree or graph data structures. It starts at the tree root (or some arbitrary node of a graph) and explores the neighbor nodes first, before moving to the next level neighbours." In our case, here is what it means, let's imagine we are at state 0:

We called the above states are at level 1. Then from each state(1-8), we can generate new states at level 2 by moving any possible tiles. For each level and each state, we check if the puzzle is solved, if it is solved, then we found the answer. Also each state maintains a path which tracks the movements from last state, that way we can output the path and generate the animation that will move the tiles.

Now let's talk about DFS, from Wikipedia, "DFS is an algorithm for traversing or searching tree or graph data structures. One starts at the root (selecting some arbitrary node as the root in the case of a graph) and explores as far as possible along each branch before backtracking." We used DFS to explore all possible paths by moving just one tile, then for each single position in the path we generate a new state.

There is another important factor we need to take into account, how to avoid duplicated or visited state? For example, from state 1 and state 3, by moving the green tile we can get the following state:
